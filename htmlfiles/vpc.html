Virtual Private Cloud (VPC) is a virtual network that I would opetate in my own data center. To create VPS with private and public subnets (smaller networks within a network), I first logged in to my AWS Management Console as an IAM user and specified Classless Inter-Domain Routing for my VPC. Classless Inter-Domain Routing (CICD) gives you flexibility in assigning IP address and routing data between devices, and VPC uses CIDR IP addresses when it transfers data packets between connected devices. Click here for more on a subnet. I then created private public subnets (smaller networks within a network) in US East (Ohio) and US East (N. Virginia) Availability Zones since AWS makes it clear that "if you host all your DB instances in one location that is affected by a failure, none of your DB instances will be available."

I then created an internet gateway , which allows communication beteween my VPC and the internet, and attached it to my VPC. Within VPC, I created three subnets (Public Subnet, Private App Subnet, and Private Data Subnet) for each of the two Availability Zones, each with a unique CICD. After making a route table called Public Route Table, next step is to add a route to the route table. This route should direct internet-bound traffic to the internet gateway just created, so I chose the destination of the route to be 0.0.0.0/0 so that any system request could be accepted. (More on that from here .) A subnet that is associated with a route table with a route to the internet gateway is called a public subnet. I associated the Public Subnets in each Availability Zone to the route. Private App Subnets and Private Data Subnets are by default associated with a Main Route Table, which is automatically created when a VPC is created. More about configuring route tables is on Configure route tables.
